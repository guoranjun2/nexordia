# Task: Refactor AIChatPanel
- **Scope:** Reduce the responsibilities of `AIChatPanel` by extracting message rendering, message history formatting, and service wiring into focused collaborators without changing user-visible behavior.
- **Motivation:** The panel currently combines view construction, message formatting, markdown conversion, and chat service setup. Separating these responsibilities will make changes safer and easier to reason about.
- **Research:**
  - `AIChatPanel` builds the user interface layout, initializes chat service wiring, formats message content, and inserts rendered output into a `JEditorPane` using `HTMLEditorKit`.
  - Assistant messages are rendered with the markdown library and fall back to escaped plain text when rendering fails.
  - Tool call summaries and user messages are still rendered as escaped plain text with newline replacement.
- **Design:**
  - Extract a dedicated message rendering component responsible for converting assistant text to HyperText Markup Language and for formatting plain text for other message categories.
  - Extract a message history component responsible for appending rendered markup to the `JEditorPane` and scrolling behavior.
  - Keep the panel responsible for layout, event wiring, and calling the new components.
  - Preserve message styling classes and existing behavior for error handling and fallback formatting.
- **Test specification:**
  - Verify assistant markdown still renders headings, lists, and emphasis in the chat history.
  - Verify non-assistant message categories still escape and preserve line breaks.
  - Verify tool call summaries still appear when enabled.
