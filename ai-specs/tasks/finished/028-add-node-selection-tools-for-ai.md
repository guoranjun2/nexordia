# Task: Add node selection tools for ai
- **Scope:** Add tools to query the full set of selected nodes and to allow ai driven selection with user visible feedback.
- **Motivation:** The model needs access to all selected nodes and the ability to select nodes to support editing workflows.
- **Research:**
  - Review how multi selection is represented in the current map view and how selections are applied.
  - Review how selection changes are surfaced to the user interface.
  - `IMapSelection.getOrderedSelection()` returns the selection list in selection order, backed by the `MapView` selection list.
  - `IMapSelection.getSortedSelection(boolean differentSubtrees)` sorts selected nodes by map order (`NodeRelativePath`), and when `differentSubtrees` is true it collapses selections so only unique subtree roots are returned.
  - `MapViewController.displayOnCurrentView(NodeModel)` delegates to `MapView.display(node)` which ensures the node is visible by restoring the view root and unfolding parent nodes as needed.
  - `ControllerProxy.select(Node)` calls `MapController.displayNode(node)` and then `selectAsTheOnlyOneSelected(node)` to select a single node and make it visible.
- **Design:**
  - Extend the existing selection identifiers tool to accept a request option for which selection collection to return. Model it as an enum with four values and default to `ORDERED`:
    - `SINGLE` (current behavior, returns only the primary selected node)
    - `ORDERED` (uses `getOrderedSelection()`)
    - `SORTED` (uses `getSortedSelection(false)`)
    - `SORTED_UNIQUE_SUBTREES` (uses `getSortedSelection(true)`)
  - Always return counts for total selected nodes and unique selected subtrees so callers can decide when to request different selection modes.
  - Return identifiers and short texts for selected nodes to reduce the need for follow up reads.
  - Hint: treat the counters as mandatory response fields across all selection modes so tests can assert them consistently.
  - Provide a tool that allows ai driven selection of a single node only, mimicking `ControllerProxy.select(Node)` by calling `MapController.displayNode(node)` and then `selectAsTheOnlyOneSelected(node)` so the selected node becomes visible in the current view (jump-in/folding aware), without changing filters.
  - Clarify in tool descriptions that selection is based on the full map model, but visibility depends on the current view (filters, folding, or jump-in), and the ai selection tool only selects a single node.
- **Test specification:**
  - Verify the selection query tool returns all selected node identifiers and includes counts for total selected nodes and unique selected subtrees.
  - Verify the ai driven selection tool updates selection in the map and is visible to the user.
