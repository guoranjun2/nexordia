# 0.1.0 Node Drag Free Position

## Scope

This development session improves node dragging around free nodes and folding-control node creation.

Main changes:

- Dragging a non-free node onto blank map space converts it to a free node and centers it on the drop point.
- Dragging an existing free node updates its position during drag instead of waiting for drop completion.
- Free-node drag keeps ordinary node drop interaction available when hovering over another node.
- The folding control can create free nodes from root and non-root nodes.
- Folding-control drag preview uses edge-style geometry for blank-space creation.
- Dragging the folding-control preview onto an existing node creates a connector and previews the connector shape that will be created.
- New free nodes inherit the parent edge style when that style differs from the default.
- Connector preview and created connectors use calculated inclinations based on source and target geometry.
- Free-node drag updates related non-`EDGE_LIKE` connector inclinations during drag.
- Folding-control scale and rotation gestures moved to modifier-based trackpad and wheel handling:
  - Control + wheel or magnification scales free child nodes.
  - Shift + magnification rotates free child nodes.

## Implementation Notes

The original UI behavior mixed two different drag flows:

- Swing node drag/drop for moving or connecting existing nodes.
- Folding-control drag for creating a child/free node or connector.

The implementation keeps those flows separate. Free-node live movement only runs while the dragged node is not over another node. When it is over another node, the map/node drop handlers continue to handle the interaction.

Free-node positioning now reuses `FreeNodePlacement` and `FreeNodeCentering`, so the same coordinate conversion is used by:

- map drop conversion to a free node,
- free-node live dragging,
- folding-control free-node creation.

Connector geometry is handled by `ConnectorInclination`. It calculates start and end inclinations from unscaled source/target bounds, then applies the result to normal connector shapes. `EDGE_LIKE` connectors are left to `EdgeLinkView` and `EdgeView` so simulated edge connectors keep native edge rendering.

For folding-control drag previews:

- blank-space previews use edge-style rendering based on the source node edge style;
- target-node previews use connector shape rendering when the connector is not `EDGE_LIKE`;
- created connectors are not forced to an explicit shape, so default and style-based connector shape settings still apply.

## Bugs Fixed

- Free nodes used to visually lag behind the pointer and then snap into place after drop.
- Some drag implementations produced a temporary rectangle-like drag shadow for free nodes.
- Live updates could break the ability to drop a dragged free node onto another node.
- Large parent nodes made folding-control-created free node placement inaccurate.
- Connector preview endpoints were previously based too much on node centers instead of edge/connector exit geometry.
- Connector previews could disagree with the connector shape created after drop.

## Risk And Caveats

- Free-node live connector updates deliberately avoid creating undo entries on every drag event.
- Connector inclination recalculation applies only to visible, non-self, non-`EDGE_LIKE` connectors during live free-node movement.
- `EDGE_LIKE` connector behavior depends on existing `EdgeLinkView` and `EdgeView` rendering.
- Swing drag/drop event timing differs between mouse and trackpad input, so manual UI verification is still important for macOS gestures.
