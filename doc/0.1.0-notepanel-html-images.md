# 0.1.0 NotePanel HTML 图片加载与宽度适配

## 背景

NotePanel 仍然使用 Swing/SimplyHTML 的 HTML 组件显示和编辑节点备注。它可能会被后续实现取代，但当前版本里依然会在打开节点备注时解析 HTML。

本次排查的问题有两个：

- 备注 HTML 中存在大量图片，或图片 URL 指向慢速/不可达网络资源时，NotePanel 可能在加载节点内容时卡住。
- NotePanel 宽度被拖窄后，备注里的大图仍保持原始宽度，导致横向撑开和反复布局。

要求是修复只作用于 NotePanel 正在使用的路径；如果 NotePanel 没显示，或者未来改成其它备注面板，不能影响普通 HTML 节点、MainView、DetailsView 等渲染路径。

## 环境

- 仓库：`/Users/peter/dev/nexordia`
- 分支：`petersun/fix/notepanel-image-resize`，合并到 `main`
- Java：JBR 21.0.7
- 构建命令：按仓库约定使用 `gradle`

## 根因

NotePanel 的查看路径原来使用 `ScaledEditorKit`。这个路径可以缩放图片，但图片 view 仍更接近 Swing 默认 HTML 图片加载行为；当 HTML 中包含很多图片或网络图片不可用时，加载和布局更容易被图片读取拖住。

NotePanel 的编辑路径来自 SimplyHTML 的 `SHTMLEditorPane`。它内部仍使用默认 SimplyHTML view factory 创建图片 view，没有复用前面为普通 HTML 节点优化过的 `LazyScaledImageView`。

另一个细节是 base URL 设置时机。旧代码在内容加载完成后才调用 `updateBaseUrl`，相对图片地址已经在文档解析阶段错过了正确 base。

宽度适配方面，之前没有一个 NotePanel 专用的最大图片宽度约束。直接改全局 HTML 图片 view 会影响脑图节点渲染，所以需要把约束挂在当前 `Document` 上，由 NotePanel 显式启用。

## 实现

新增 `HtmlImageViewSettings`：

- 用 HTML `Document` 属性保存最大图片宽度。
- 用 HTML `Document` 属性保存图片 view 的缩放倍率。
- 用 `src-width` 和 `src-height` 保存图片源尺寸，作为响应式缩放的上限；该写入只发生在 NotePanel 加载内容后，不在 HTML view 布局过程中写文档。
- 仅当图片宽度超过文档设置的最大宽度，且图片自身宽度大于 100px 时，才等比缩小；缩小后的宽度不低于 100px。
- 默认没有文档属性时保持原行为。

新增 `LazyImageSHTMLEditorKit`：

- 给 SimplyHTML 编辑器使用。
- 非 GIF 图片使用 `LazyScaledImageView`。
- GIF 图片继续使用 `ScaledGifImageView`，保留 GIF 动画路径。

调整 `NotePanel`：

- HTML 查看面板从 `ScaledEditorKit` 改为 `SynchronousScaledEditorKit`，复用懒加载图片 view。
- SimplyHTML 编辑器替换为 `LazyImageSHTMLEditorKit`，同时保留原有 `SHTMLDocument`，避免破坏 SimplyHTML 面板内部状态。
- 在加载查看/编辑内容前先给当前文档写入 NotePanel 图片设置。
- 加载内容和面板 resize 时都按当前可见宽度写入图片宽度上限；拖拽过程中图片可以实时变小或变大，但不会超过图片自身宽度，且大图不会被压到 100px 以下。
- resize/change 监听同时挂在 NotePanel、viewer viewport、editor pane 和 editor viewport 上，避免 split pane 或窗口拖拽只改变内部可见区域时漏掉收缩。
- 宽度变小时同步调整 editor pane 的实际宽度，避免 Swing 在图片曾经撑宽内容后继续沿用旧组件宽度。
- 图片宽度属性变化后会显式通知 HTML root view 和图片 view 的 preference 失效，避免 Swing text view 树继续使用旧的 preferred/minimum span。
- NotePanel 文档中的图片如果已有 `width` / `height` 且缺少源尺寸属性，会写入 `src-width` / `src-height`；普通 HTML 节点渲染没有启用 NotePanel 文档属性，不会写这些属性。
- `imageMaximumWidth` 在 `NotePanel.isShowing()` 为 false 时返回 0，所以 NotePanel 未显示时不会启用宽度限制。
- HTML 查看内容加载前重建 viewer document，避免旧文档里的图片状态和 base URL 影响新节点。

调整 `NoteManager`：

- 在读取和设置备注内容前调用 `notePanel.updateBaseUrl(node.getMap().getURL())`。
- 保证相对图片地址在 HTML 文档解析前拿到正确 base。

## 放弃或避免的方案

- 没有把最大图片宽度做成全局设置。这样会影响 MainView、DetailsView 和其它 HTML 渲染路径，不符合 NotePanel 可替换的边界。
- 没有在 NotePanel 隐藏时继续维护图片宽度。隐藏状态下没有可靠 viewport 宽度，而且用户明确要求不使用 NotePanel 时不要有影响。
- 没有为网络图片增加新的同步预读取逻辑。同步读取会重新引入卡顿风险；本次复用已有懒加载图片 view。

## 测试覆盖

新增和扩展的 focused tests：

- 宽图在文档设置最大宽度时等比缩小。
- 小于或等于 100px 的图片即使超过面板宽度也保持原始尺寸。
- 大图在面板宽度小于 100px 时最多缩到 100px。
- 文档级图片 zoom 设置会覆盖默认 map zoom。
- SimplyHTML 编辑器图片 view 使用 `LazyScaledImageView`。

## 剩余风险

- 这次没有做手工 UI 回归，只通过 HTML view 层的 focused tests 和 `freeplane` 测试集验证。
- NotePanel 编辑器是 SimplyHTML 封装组件，替换 editor kit 时保留了原文档；如果 SimplyHTML 后续版本对 editor kit 和 document 的绑定假设更严格，需要重新检查。
- GIF 仍使用 Swing 的 GIF 图片路径，只新增最大宽度适配，没有改 GIF 网络加载模型。
