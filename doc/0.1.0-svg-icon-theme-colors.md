# 0.1.0 SVG 图标主题色适配

日期: 2026-06-26

## 背景

工具栏、偏好面板和节点布局下拉面板中有大量 UI SVG 图标使用 `?useAccentColor=true`。在日间主题下这些图标偏淡，在夜间主题下又可能继续保持旧颜色或过暗。用户通过右下角主题按钮切换主题后，已存在的图标实例没有稳定跟随新主题更新。

侧边栏工具窗口图标提供了一个稳定参考：`ModernToolWindowPanel.ToolWindowIcon` 不是预先渲染好的图片，而是在 `paintIcon(...)` 中读取当前组件的 `foreground` 后直接绘制线条。因此它不需要清理缓存，也不依赖主题切换事件顺序；只要组件前景色被 Look and Feel 更新，下一次 repaint 就会使用新颜色。

## 根因

SVG 图标旧路径有多个会让颜色固定住的缓存层：

1. `SVGIconCreator` 以前用静态字段缓存 `uiColorReplacements`。该替换表第一次生成后不会随主题切换重新计算。
2. SVGSalamander 的 `SVGUniverse` 会按内部 URI 复用已经加载的 SVG diagram。即使替换表变化，如果 URI 不变，仍可能拿到旧颜色 diagram。
3. `CachingIcon` 首次生成真实 `ImageIcon` 后会丢弃 supplier。
4. 工具栏构建代码会把部分 `Icon` 转成 `ImageIcon`，颜色因此被烤进像素里。

前面尝试过清理缓存、刷新 action icon、重载 ResourceController 图标缓存等方向，但这类方案依赖主题切换时机和组件刷新范围，仍然容易遗漏已经持有旧 `ImageIcon` 的组件。

## 最终方案

最终采用和侧边栏一致的思路：带 `?useAccentColor=true` 的 UI SVG 不再只按当前 Look and Feel 生成一次静态图，而是通过 `ThemedSVGIcon` 在绘制时读取组件当前前景色。

实现要点：

- `FreeplaneIconFactory.createSVGIcon(...)` 只对 `?useAccentColor=true` 的 SVG 返回 `ThemedSVGIcon`。
- `ThemedSVGIcon.paintIcon(...)` 根据传入组件的 `foreground` 生成替换规则。
- `#333`、`#333333`、`#000`、`#000000` 和旧蓝色 `#0071bc` 映射为当前组件前景色。
- `#999`、`#999999` 映射为禁用前景色，优先使用 `Button.disabledText`、`Label.disabledForeground`、`Slider.disabledTrackColor`。
- 同一个 `ThemedSVGIcon` 内部只缓存少量按颜色区分的 SVG delegate，避免每次 paint 都重新解析 SVG。
- `SVGIconCreator` 支持传入显式替换规则，并把替换规则 hash 加入 SVGUniverse 内部 URI，避免不同主题颜色复用同一个 diagram。
- 普通 SVG、节点内容图标和不带 `useAccentColor=true` 的彩色语义图标仍走原路径。

这样主题切换后不需要显式刷新图标缓存。Look and Feel 更新组件前景色后，下一次 repaint 会自然得到新颜色。

## 测试覆盖

新增 `ThemedSVGIconTest` 覆盖两个关键行为：

1. 同一个图标实例先用深色前景色绘制，再把组件前景色改成浅色后重绘，像素颜色随之变化。
2. 中性颜色 `#333333` 也会映射为当前组件前景色。

该测试直接覆盖“主题切换后已有图标实例继续使用旧颜色”的核心问题。

## 剩余风险

`ThemedSVGIcon` 只处理已有 UI SVG 中常见的中性和旧蓝色颜色值。如果后续引入新的固定色值，并且也希望它跟随主题变化，需要把对应颜色加入替换规则，或进一步把 SVG 资源规范为统一的 current-color 占位色。

如果某个 `?useAccentColor=true` SVG 实际上希望保留蓝色语义色，本次会把 `#0071bc` 映射为组件前景色。这符合当前“UI 图标随主题前景色变化”的目标，但新增语义彩色图标时应避免使用 `useAccentColor=true`。
