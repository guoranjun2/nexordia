# 0.1.0 低缩放节点渲染优化

## 背景

大图在低缩放比例下拖拽视图时容易卡顿。问题特征是：缩放率较大、同屏节点数量较少时拖拽较流畅；缩放率较小、同屏节点数量暴涨时拖拽卡顿明显。

这说明瓶颈主要来自低缩放下大量可见节点的重复绘制，而不是单个节点尺寸较大时的绘制成本。

## 本次优化

优化集中在节点内容绘制入口：

- `MainView.paintComponent`
- `DetailsView.paintComponent`
- `ZoomableLabel`

当节点在当前 `MapView` 可见区域外，或者在极低缩放下节点已经小到无法辨认内容时，跳过完整文本、HTML、图标和复杂形状绘制，改为绘制简化矩形。

简化绘制阈值集中在 `ZoomableLabel` 中：

- `SIMPLIFIED_PAINTING_MAX_ZOOM = 0.2f`
- `SIMPLIFIED_PAINTING_MAX_WIDTH = 10`
- `SIMPLIFIED_PAINTING_MAX_HEIGHT = 3`

## 可见性判断

曾尝试用 `getParent().contains(getLocationOnScreen())` 判断节点是否可见。这个方式在局部场景中有效，但后来发现 parent 中可能存在其他组件，导致宽高或坐标偏移，右侧、下侧边缘露出的节点可能被误判为不可见。

最终改为直接以 `MapView` 为基准：

1. 使用 `getLocationOnScreen()` 取得组件屏幕坐标。
2. 使用 `SwingUtilities.convertPointFromScreen(point, map)` 转换到 `MapView` 坐标。
3. 使用 `map.getVisibleRect()` 判断节点四角是否有任一点可见。
4. 右侧和下侧采样点使用 `width - 1`、`height - 1`，避免落到排他边界之外。

这个判断覆盖以下边缘场景：

- 节点从左侧露出右边缘。
- 节点从右侧露出左边缘。
- 节点从上方露出下边缘。
- 节点从下方露出上边缘。

## 相关实现

`ZoomableLabel.shouldPaintSimplified(int width, int height)` 统一判断是否需要简化绘制。

`ZoomableLabel.paintSimplified(Graphics graphics, int width, int height, Color borderColor)` 统一绘制简化矩形。

`MainView` 使用节点背景色和边框色绘制简化节点。

`DetailsView` 使用同一套可见性和尺寸判断，避免低缩放下细节文本继续走完整绘制路径。

## 后续观察

如果低缩放拖拽仍然卡顿，下一步应继续分析整体 repaint pass 的成本，尤其是：

- 同屏大量节点的边和连接线绘制。
- 云框绘制。
- `MapView.paintChildren` 多轮 painting mode。
- 拖拽过程中的 repaint 频率和事件合并。
