# 0.1.0 macOS 编译记录

## 目标

构建 Freeplane 0.1.0 的 macOS 安装程序，要求安装程序内置 Java 运行环境，不依赖用户本机预装 Java。

## 环境

- 仓库：`/Users/peter/src/freeplane`
- 系统：macOS 26.4.1，arm64
- Gradle：`/opt/homebrew/bin/gradle`，版本 9.5.0
- 实际使用 Java：Homebrew OpenJDK 21.0.11
- `JAVA_HOME`：`/opt/homebrew/opt/openjdk@21/libexec/openjdk.jdk/Contents/Home`
- `jlink`：21.0.11
- `jpackage`：21.0.11
- 代码签名身份：无可用身份，`security find-identity -v -p codesigning` 返回 `0 valid identities found`

仓库说明中要求优先使用 `/Users/peter/.sdkman/candidates/java/21.0.5-zulu`，但该路径在本机不存在，因此改用本机可用的 OpenJDK 21.0.11。

## 最终产物

- 通用发行 zip：`DIST/freeplane_bin-0.1.0.zip`，约 67M
- macOS 安装程序：`DIST/Freeplane-0.1.0.dmg`，约 125M
- 中间 app bundle：`build/macosapp/app-image-arm64/Freeplane.app`，约 236M
- 内置运行时：`build/macosapp/runtime-arm64`，约 153M

最终 DMG 是未签名版本。由于本机没有 Developer ID 证书，无法完成代码签名和 notarization。分发给其他机器时可能触发 Gatekeeper 提示。

## 尝试过程

### 1. 环境探测

检查了 Java、Gradle、`jpackage` 和签名身份：

```bash
/usr/libexec/java_home -V
java -version
which java
which gradle
gradle --version
security find-identity -v -p codesigning
```

发现：

- `/Users/peter/.sdkman/candidates/java/21.0.5-zulu` 不存在。
- `/opt/homebrew/opt/openjdk@21/bin/java` 可用。
- `jpackage` 和 `jlink` 可用。
- 没有可用 codesign identity。

### 2. 官方 Gradle macOS 任务不可直接使用

查看 `mac.dist.gradle` 后发现，macOS 打包任务通过 `defineMacBuildTasks('intel')` 和 `defineMacBuildTasks('apple')` 动态创建，并且外层条件要求：

```groovy
Os.isFamily(Os.FAMILY_MAC) && macAppProperties['macapp.codesign.identity'] != null
```

也就是说，如果没有 `macapp.properties` 中的 `macapp.codesign.identity`，官方 `dmg4mac_apple` / `dmg4mac_intel` 路径不会正常创建。由于本机没有签名身份，因此没有走官方签名打包任务，而是复用同一套 `jpackage` 参数手工构建未签名 DMG。

### 3. 第一次构建失败：翻译格式校验

首次执行：

```bash
export JAVA_HOME="/opt/homebrew/opt/openjdk@21/libexec/openjdk.jdk/Contents/Home"
export PATH="$JAVA_HOME/bin:$PATH"
gradle clean binZip
```

构建失败在：

```text
Task :freeplane:check_translation FAILED
Resources_nl.properties requires formatting - content changed
```

解决方式：

```bash
gradle format_translation
```

该命令修改了 `freeplane/src/editor/resources/translations/Resources_nl.properties`，主要是把新增 AI 相关 key 移到排序位置，并把非 ASCII 字符转成 properties 文件要求的转义形式。

### 4. 第二次构建成功生成发行 zip

再次执行：

```bash
gradle binZip
```

成功生成：

```text
DIST/freeplane_bin-0.1.0.zip
```

过程中出现大量 Java 8 target 在新 JDK 下的弃用警告，以及 Javadoc 报告中的错误文本，但 `binZip` 任务最终成功。

### 5. 生成内置 Java runtime

执行：

```bash
rm -rf build/macosapp/runtime-arm64
jlink \
  --add-modules ALL-MODULE-PATH \
  --strip-debug \
  --no-man-pages \
  --no-header-files \
  --output build/macosapp/runtime-arm64
```

输出运行时约 153M。`jlink` 打印：

```text
WARNING: Using incubator modules: jdk.incubator.vector
```

该警告来自当前 JDK 模块集，不影响 runtime image 生成。

### 6. 准备 jpackage 输入目录

执行：

```bash
rm -rf build/macosapp/zip-extracted build/macosapp/input build/macosapp/app-image-arm64 build/macosapp/dmg-resource-dir
mkdir -p build/macosapp/zip-extracted build/macosapp/input build/macosapp/dmg-resource-dir
unzip -q DIST/freeplane_bin-0.1.0.zip -d build/macosapp/zip-extracted
cp build/macosapp/zip-extracted/freeplane-0.1.0/*.* build/macosapp/input/
cp freeplane_framework/mac-appbundler/freeplane.icns build/macosapp/dmg-resource-dir/Freeplane-volume.icns
```

这里先只复制顶层文件到 `build/macosapp/input`，与 `mac.dist.gradle` 中 `macosapp_input` 的行为保持一致。完整目录内容后续再复制进 app bundle。

### 7. 生成 `.app`

使用仓库 `mac.dist.gradle` 中的主要 `jpackage` 参数生成 app image：

```bash
jpackage \
  --name Freeplane \
  --type app-image \
  --icon freeplane_framework/mac-appbundler/freeplane.icns \
  --app-version 0.1.0 \
  --java-options "-XX:-IgnoreUnrecognizedVMOptions" \
  --java-options "-XX:MaxRAM=20g" \
  --java-options "-XX:MaxRAMPercentage=15.0" \
  --java-options "-Djava.security.manager=allow" \
  --java-options "-Dapple.laf.useScreenMenuBar=false" \
  --java-options "-Xdock:name=Freeplane" \
  --java-options "--add-exports" \
  --java-options "java.desktop/com.apple.eawt=ALL-UNNAMED" \
  --java-options "--add-exports" \
  --java-options "java.desktop/com.apple.eawt.event=ALL-UNNAMED" \
  --java-options "--add-exports" \
  --java-options "java.desktop/sun.awt=ALL-UNNAMED" \
  --java-options "--add-exports" \
  --java-options "java.desktop/sun.swing=ALL-UNNAMED" \
  --java-options "--add-exports" \
  --java-options "java.desktop/com.apple.laf=ALL-UNNAMED" \
  --java-options "--add-opens" \
  --java-options "java.desktop/javax.swing.text.html=ALL-UNNAMED" \
  --input build/macosapp/input \
  --main-class org.freeplane.launcher.Launcher \
  --main-jar freeplanelauncher.jar \
  --runtime-image build/macosapp/runtime-arm64 \
  --copyright "Freeplane 0.1.0" \
  --dest build/macosapp/app-image-arm64
```

该步骤成功生成：

```text
build/macosapp/app-image-arm64/Freeplane.app
```

### 8. 补齐 app bundle 内容

与 `mac.dist.gradle` 中 `macosapp_makeapp` 的做法一致，把完整发行内容复制进 app bundle：

```bash
cp -R build/macosapp/zip-extracted/freeplane-0.1.0/. build/macosapp/app-image-arm64/Freeplane.app/Contents/app/
cp -R freeplane_framework/mac-appbundler/freeplanedoc.icns build/macosapp/app-image-arm64/Freeplane.app/Contents/Resources/
```

然后用仓库提供的 XSLT 改写 `Info.plist`：

```bash
cp build/macosapp/app-image-arm64/Freeplane.app/Contents/Info.plist build/macosapp/app-image-arm64/Freeplane.app/Contents/Info.plist.orig
xsltproc freeplane_framework/mac-appbundler/mac_info_plist.xslt \
  build/macosapp/app-image-arm64/Freeplane.app/Contents/Info.plist.orig \
  > build/macosapp/app-image-arm64/Freeplane.app/Contents/Info.plist
rm build/macosapp/app-image-arm64/Freeplane.app/Contents/Info.plist.orig
```

`xsltproc` 输出过一个 Apple DTD 加载警告：

```text
warning: failed to load external entity "https://www.apple.com/DTDs/PropertyList-1.0.dtd"
```

转换结果仍然生成成功。

### 9. 生成 DMG

执行：

```bash
jpackage \
  --name Freeplane \
  --type dmg \
  --app-version 0.1.0 \
  --app-image build/macosapp/app-image-arm64/Freeplane.app \
  --mac-package-identifier org.freeplane.core \
  --mac-package-name Freeplane \
  --resource-dir build/macosapp/dmg-resource-dir \
  --dest DIST
```

成功生成：

```text
DIST/Freeplane-0.1.0.dmg
```

中间有一次清理旧 DMG 的命令使用了通配符，因没有匹配项，zsh 输出：

```text
zsh: no matches found: DIST/Freeplane-*.dmg
```

该提示来自 shell 通配符展开失败，不影响后续 `jpackage` 执行，也没有阻止 DMG 生成。

## 验证

检查产物：

```bash
find DIST -maxdepth 1 -type f -print -exec ls -lh {} \;
du -sh build/macosapp/app-image-arm64/Freeplane.app \
      build/macosapp/app-image-arm64/Freeplane.app/Contents/runtime \
      build/macosapp/app-image-arm64/Freeplane.app/Contents/app
```

结果：

```text
DIST/freeplane_bin-0.1.0.zip    67M
DIST/Freeplane-0.1.0.dmg        125M
Freeplane.app                    236M
Contents/runtime                 155M
Contents/app                      80M
```

执行签名相关检查：

```bash
codesign --verify --deep --strict --verbose=2 build/macosapp/app-image-arm64/Freeplane.app
spctl -a -vv build/macosapp/app-image-arm64/Freeplane.app
```

结果：

```text
build/macosapp/app-image-arm64/Freeplane.app: invalid Info.plist (plist or signature have been modified)
In architecture: arm64
build/macosapp/app-image-arm64/Freeplane.app: accepted
override=security disabled
```

解释：

- `codesign` 报错是预期风险：app 未签名，并且 `Info.plist` 在 `jpackage` 生成后被 XSLT 修改。
- `spctl` 显示 accepted 是因为当前机器安全策略关闭，不代表可直接分发。

## 遗留问题

1. 该 DMG 未签名、未 notarize，不适合作为正式公开发布包。
2. 如果要走官方 `mac.dist.gradle` 流程，需要准备 `macapp.properties`，并提供可用的 Developer ID 签名身份和 notarization keychain profile。
3. 当前产物为 arm64 runtime，适合 Apple Silicon。若需要 Intel 版本，需要使用 x64 JDK 和 x64 runtime image 重新构建。
4. `gradle format_translation` 修改了 `Resources_nl.properties`，需要在提交前确认该格式化改动是否应该随本次工作保留。

## 后续建议

正式发布前建议使用官方 Gradle macOS 任务，在具备签名证书的机器上完成：

```bash
gradle clean binZip dmg4mac_apple
gradle notarytool_apple
```

如果需要同时发布 Intel 版本，还需要配置 `macapp.intel.jpackage.jdk` 和 `macapp.intel.jdk.runtime.image`，再执行对应的 Intel 任务。

## 2026-06-01 补充：新增一键未签名 DMG 任务

### 背景

IntelliJ IDEA 的 Gradle 工具窗口中已有顶层 `build` 等任务，但没有一个无需签名配置即可直接生成 macOS DMG 的任务。仓库原本的 macOS 打包逻辑在 `mac.dist.gradle` 里，官方任务包括 `dmg4mac_apple` / `dmg4mac_intel` / `notarytool_apple` / `notarytool_intel`，但这些任务只有在满足以下条件时才会动态创建：

```groovy
Os.isFamily(Os.FAMILY_MAC) && macAppProperties['macapp.codesign.identity'] != null
```

也就是说，原仓库有正式发布用的签名打包任务，但没有本机开发用的“无证书一键生成未签名 DMG”任务。本机没有 codesign identity，因此官方任务不适合当前开发调试流程。

### 修改

在 `mac.dist.gradle` 中新增以下 Gradle 任务：

- `macUnsignedRuntime`
- `macUnsignedInput`
- `macUnsignedAppImage`
- `macUnsignedApp`
- `macUnsignedDmg`

其中最终入口是：

```bash
gradle macUnsignedDmg
```

该任务会：

1. 使用当前 Gradle JVM 所在 JDK 的 `jlink` 生成内置 runtime。
2. 依赖 `binZip` 生成 Freeplane 二进制发行 zip。
3. 解压发行 zip 并准备 `jpackage` 输入目录。
4. 使用 `jpackage --type app-image` 生成未签名 `.app`。
5. 补齐 `core/`、`plugins/`、`doc/`、`resources/`、`scripts/` 等完整应用内容。
6. 使用仓库已有 `mac_info_plist.xslt` 更新 `Info.plist`。
7. 使用 `jpackage --type dmg` 输出未签名 DMG。

### 验证

先验证任务是否出现在 Gradle 任务列表：

```bash
gradle tasks --all | rg "macUnsigned|distribution" -n
```

确认输出包含：

```text
macUnsignedApp
macUnsignedAppImage
macUnsignedDmg
macUnsignedInput
macUnsignedRuntime
```

再验证任务依赖链：

```bash
gradle macUnsignedDmg --dry-run
```

依赖链成功通过，最终包含：

```text
:binZip SKIPPED
:macUnsignedInput SKIPPED
:macUnsignedAppImage SKIPPED
:macUnsignedApp SKIPPED
:macUnsignedDmg SKIPPED
BUILD SUCCESSFUL
```

最后实际执行：

```bash
gradle macUnsignedDmg
```

构建成功，耗时约 3 分钟，最终输出：

```text
DIST/Freeplane-0.1.0-apple-unsigned.dmg
```

文件大小约 128M。

### IntelliJ 使用方式

刷新右侧 Gradle 面板后，可以在顶层项目任务里找到：

```text
freeplane_root
  > Tasks
    > distribution
      > macUnsignedDmg
```

双击 `macUnsignedDmg` 即可一键生成未签名、内置 Java runtime 的 macOS DMG。

### 注意事项

1. 该任务使用当前运行 Gradle 的 JDK，因此 Gradle 必须由完整 JDK 启动，不能是 JRE。
2. 该任务适合本地开发、测试和内部安装验证，不替代官方签名发布流程。
3. Apple Silicon 机器上输出 `*-apple-unsigned.dmg`，Intel 机器上会输出 `*-intel-unsigned.dmg`。
4. 正式分发仍应使用带签名和 notarization 的官方任务。
