# 1.13.3 Background Image Notes

日期: 2026-06-16

## 背景

这次工作围绕导图背景图片体验展开，目标是让背景图片更适合长期作为地图底图使用，同时保留 Freeplane 原有 map style 属性模型。

涉及的用户需求包括:

- 默认启用背景图显示。
- 默认启用 `Fit background image to Window`。
- fit 模式按视口等比例覆盖，超出部分居中裁剪。
- 增加临时禁用背景图的开关，不删除原背景链接。
- 增加背景图透明度设置。
- macOS 选择背景图片时使用系统文件选择器，并允许显示隐藏目录。
- 背景图片选择器默认从系统 Pictures 目录打开；如果当前已有本地背景图片，优先打开其所在目录。
- 支持网络图片 URL，并把远程图片缓存到用户 resources 目录。
- 试验性支持 GIF 背景图动画播放。

## Map Style 属性

背景图仍使用原有 map style 属性 `backgroundImageURI` 保存图片 URI。

新增或调整的相关属性:

- `background_image_enabled`: 是否显示背景图片。默认 `true`。关闭时只隐藏背景图，不清除 `backgroundImageURI`。
- `fit_to_viewport`: 是否把背景图适配到窗口。默认 `true`。内置模板和 `xml2mm.xsl` 中显式默认值也同步改为 `true`。
- `background_image_opacity`: 背景图片透明度百分比，范围 `0` 到 `100`，默认 `100`。

这些属性都通过现有 `MapStyle` / `MapStyleModel` 机制保存和触发地图刷新。

## Fit 模式

`fit_to_viewport=true` 时，背景图不再简单拉伸到视口尺寸，而是采用等比例覆盖:

- 先按图片原始比例缩放到覆盖整个视口。
- 如果缩放后宽度或高度超出视口，超出部分居中裁剪。
- 图片比例不被改变。

实现上，`BitmapViewerComponent` 在 `cropsToViewerSize=true` 时计算覆盖尺寸。绘制时用视口尺寸作为裁剪参考，而不是用缩放后的图片尺寸作为参考，这样裁剪中心保持在图片中心。

普通非 fit 模式仍使用原有逻辑: 背景图跟随地图缩放，并以根节点中心为锚点绘制。

## 透明度

`background_image_opacity` 放在 Preferences 的 Painting tab 下，分组为 `background_image`。

透明度通过 `AlphaComposite.SRC_OVER` 应用于背景图片绘制阶段:

- 非 fit 背景图在 `MapView.paintBackgroundComponent()` 中设置 composite。
- fit 背景图在 `MapViewScrollPane.MapViewPort.paintComponent()` 中设置 composite。

这只影响背景图片，不影响地图背景色、节点、连线、坐标轴或选择框。

修改透明度配置时，当前地图和 viewport 会触发 repaint。

## macOS 文件选择器

背景图片选择仍走 Freeplane 的文件选择器抽象，但 macOS 上会使用系统原生文件选择器。

相关行为:

- 设置 `apple.awt.fileDialogShowHiddenFiles=true`，让隐藏文件和隐藏目录可见。
- Swing fallback 下关闭 file hiding。
- 背景图片选择默认目录为 `~/Pictures`，如果不存在则回退到用户主目录。
- 如果当前导图已有本地背景图片，下一次选择背景图片时优先打开该图片所在目录。

曾经尝试过记录上次选择目录，但最终删除，避免和 Freeplane 现有资源路径控制重复。

## 网络图片缓存

`backgroundImageURI` 可以保存 `http` 或 `https` 图片链接。持久化时仍保存原始 URL，不改写成缓存文件路径。

加载背景图时:

1. 如果 URI 不是 `http/https`，直接按原逻辑加载。
2. 如果 URI 是远程图片，先根据 URL 计算缓存文件名。
3. 在用户目录的 `resources/background/` 下查找缓存文件。
4. 如果缓存存在，使用本地缓存文件加载背景图。
5. 如果缓存不存在，后台请求远程 URL，当前背景图保持不变。
6. 请求成功后保存到缓存目录，再在 EDT 中用缓存文件替换背景图。
7. 请求失败时记录 warn，本次不替换背景图，也不再回退到原始远程 URL 同步读图。

远程图片加载、缓存文件命名、请求头、异步下载和过期请求保护被抽取到 `MapBackgroundImageLoader`。`MapView` 只负责读取 map style、取得 viewer factory，并在 loader 回调时安装或清除背景组件。

异步下载使用单线程 daemon executor。每次加载都会递增请求序号，下载完成后只有序号仍然匹配时才替换背景图。这样用户快速切换背景图、禁用背景图或重新加载地图时，旧下载不会覆盖当前状态。

远程请求会设置浏览器式 `User-Agent`、图片 `Accept` 和 `Accept-Language`。对 `pexels.com` 图片链接额外设置 `Referer: https://www.pexels.com/`，避免常见 CDN 直接返回 403。`Accept` 优先请求 Java 图片栈能稳定读取的 PNG/JPEG/GIF，避免服务端返回 AVIF 后本地无法解码。

缓存文件名使用:

```text
sha256(url) + 原始扩展名
```

这样可以避免 URL 中的特殊字符、中文、查询参数和超长路径问题。SHA-256 碰撞概率可以忽略。保留原始扩展名是为了让 `.gif` 缓存后仍能走 GIF 动画路径。

当前没有实现缓存失效策略。同一个 URL 的远程内容如果变化，Freeplane 会继续使用已有缓存。需要刷新时可以删除对应缓存文件，或者后续增加显式刷新命令。

曾讨论过连续三次请求失败后删除背景链接，但最后取消，没有实现。

## GIF 背景图

静态图片仍走 `ImageIO.read()`、缩放和缓存路径。

GIF 文件走单独路径:

- 根据 URI 扩展名判断 `.gif`。
- 使用 `ImageIcon` 加载 GIF，保留 Swing 自带动画帧推进。
- 绘制时仍复用背景图片的缩放、fit 和居中裁剪坐标。

第一次实现只用 `ImageIcon` 绘制，但发现只有用户滚动导图时 GIF 才会播放。原因是 fit 背景图不是普通 Swing 子组件，而是在 viewport paint 中手动调用 `backgroundComponent.paintComponent(g)`，所以 GIF 组件自己的 repaint 不会触发真正的背景绘制。

后续修正:

- `BitmapViewerComponent` 为 GIF 启动一个轻量 Swing `Timer`。
- timer 触发时优先调用 `rendererListener`，让真正绘制背景的 viewport repaint。
- 如果没有 renderer listener，再退回 repaint 自己。
- `ScalableComponent` 增加 `stopAnimation()` 默认方法。
- `MapView` 替换、禁用或清除背景组件时调用 `stopAnimation()`，避免旧 GIF 在后台继续刷新。

这个 GIF 支持仍是试验性质。它没有自己解析 GIF 帧 delay，也没有实现独立帧缓存，而是依赖 Swing `ImageIcon`。

## Groovy 脚本示例

设置本地背景图片:

```groovy
def image = new File('/Users/peter/Pictures/background.jpg')
node.map.storage['backgroundImageURI'] = image.toURI().toString()
node.map.storage['fit_to_viewport'] = 'true'
node.map.storage['background_image_enabled'] = 'true'
```

设置网络背景图片:

```groovy
node.map.storage['backgroundImageURI'] = 'https://example.com/background.gif'
node.map.storage['fit_to_viewport'] = 'true'
node.map.storage['background_image_enabled'] = 'true'
```

临时隐藏背景图片:

```groovy
node.map.storage['background_image_enabled'] = 'false'
```

重新显示背景图片:

```groovy
node.map.storage['background_image_enabled'] = 'true'
```

调整透明度:

```groovy
config.setProperty('background_image_opacity', 60)
```

清除背景链接:

```groovy
node.map.storage['backgroundImageURI'] = null
```

## 失败尝试和取舍

曾经尝试把背景图片模式改成二级菜单，包含 fit、单张、平铺和禁用模式。这个方向后来取消，回到原有 `Fit background image to Window` 菜单项，并只增加 `Activate background image` 开关。

曾经讨论根据图片是否正方形或分辨率大小自动决定平铺/整体背景。这个方向也取消，避免隐式规则带来不可预测行为。

曾经尝试记忆文件选择器目录。后来删除，背景图片选择默认使用系统 Pictures 目录，已有背景图时使用当前背景图所在目录。

GIF 动画第一次只调用组件 repaint，实际无效，因为 fit 背景图由 viewport 手动绘制。最终改为通过 renderer listener repaint 真正的承载层。

远程图片第一次实现为同步下载，打开或设置慢速网络图片时会阻塞 UI。后来改为 `MapBackgroundImageLoader` 异步下载；未缓存远程图片下载完成前保留当前背景图，下载完成后再替换。

## 验证

常用验证命令:

```bash
gradle format_translation
gradle :freeplane:compileJava
gradle :freeplane:test
git diff --check
```

人工验证建议:

- 选择本地 PNG/JPG 背景图，确认默认 fit 模式启用。
- 切换 `Fit background image to Window`，确认 fit 模式居中裁剪，非 fit 模式跟随地图缩放。
- 调整 `Background image opacity (%)`，确认只影响背景图，不影响节点和坐标轴。
- 关闭 `Activate background image`，确认背景图隐藏但链接没有被清除。
- 再次选择背景图，确认默认目录为当前背景图所在目录或 `~/Pictures`。
- 设置远程图片 URL，确认用户目录 `resources/background/` 生成缓存文件。
- 重新打开导图，确认同一 URL 使用本地缓存。
- 设置 GIF 背景图，确认不滚动地图也能播放。
