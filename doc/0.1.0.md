# Nexordia 0.1.0 Development Log

## Scope

This version establishes Nexordia as an independently maintained fork of Freeplane 1.13.x while preserving the original project history, license, copyright notices, package structure, and compatibility-oriented technical identifiers.

## Changes

- Renamed user-facing product identity from Freeplane to Nexordia in the README, application resources, English UI strings, launch scripts, installers, package metadata, and AI plugin documentation.
- Added `NOTICE.md` to record the Freeplane fork origin and Nexordia contributor copyright.
- Set the application version to `0.1.0`.
- Changed default user configuration locations to Nexordia-specific directories so Nexordia does not read or overwrite Freeplane user configuration by default.
- Removed automatic copying from old Freeplane versioned user directories during startup.
- Replaced the versioned user README resource with a Nexordia-specific `userReadme.txt`.
- Renamed distribution artifacts, Linux package metadata, Windows executable names, Windows installer labels, macOS bundle names, and script-level environment variables where they are user-facing.
- Kept internal Java packages, OSGi bundle identifiers, Gradle subproject names, and map MIME compatibility names where changing them would create unnecessary risk at this stage.
- Updated the AI MCP server display name and custom token header to use Nexordia naming.
- Changed the local Git `origin` remote to `git@github.com:guoranjun2/nexordia.git`.
- Kept `upstream` pointing to `https://github.com/freeplane/freeplane.git` for fetching upstream fixes, with its push URL disabled.

## Modern Layout And Breadcrumb Navigation

- Reworked the modern tool window layout around IDEA-style left and right button rails. The default layout keeps the outline on the left, and keeps icons, style preferences, and the note panel on the right.
- Added side panel collapse handles on the inner edge of each side panel. The handle direction follows the panel side, and collapsing a side also collapses its button pane.
- Replaced the title-only window header with a breadcrumb bar. The file breadcrumb shows the full path with folder and file icons, and the node breadcrumb follows the selected node path.
- Switched file and node breadcrumb popups back to native Swing multi-level menus. Parent menu rows support direct clicks, and child menus are loaded lazily on hover so deep paths remain responsive.
- The node breadcrumb menu defaults to the current node when opened, renders node icons from the node itself, shows blank node text as two spaces, and adds the first child preview in muted text when a node has children.
- Node breadcrumb compression is calculated dynamically from the compressed text width, keeping the visible path readable while preserving the current node.
- Added debounced node breadcrumb refresh after node changes so path text updates without repaint storms during active edits.
- Updated bookmark and breadcrumb node navigation to jump immediately, center the target node, and avoid the slow-scroll positioning drift seen at low zoom levels.
- Added bookmark behavior options for centering and optional 100% zoom reset under the Behavior preferences tab.

## Implementation Notes

- Native Swing menus were kept for file and node breadcrumbs because the custom popup experiment made submenu travel and visual hierarchy worse than the platform behavior.
- Lazy breadcrumb menus override native mouse release handling only for the parent row primary action; hover and submenu traversal still use Swing menu selection.
- Node centering now favors immediate viewport correction after selection instead of animated scrolling. Animated scrolling can accumulate stale geometry at low zoom because selection, layout invalidation, and viewport movement do not always settle in the same event turn.

## Risks And Notes

- The repository still contains many `org.freeplane.*` packages and technical identifiers by design. They remain part of the compatibility and OSGi surface until a dedicated internal rename effort is planned and tested.
- GitHub `origin/main` currently points at an unrelated initial commit. Publishing this local history as GitHub `main` requires an explicit force-with-lease push.
- Some icon filenames and installer script filenames still contain Freeplane in source paths while their generated artifacts are renamed to Nexordia.
- Translation changes were limited to English user-facing strings and installer messages. Full multilingual brand updates should be handled separately with the Java properties encoding workflow.
